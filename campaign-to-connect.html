Campaign to Connect — API Mapping Reference

Maps Acoustic Campaign APIs to their Acoustic Connect equivalents with typical use cases
REST APIs
Campaign API How It Is Being Used Connect Equivalent Notes
REST
GET Contact by Lookup Key		 Look up a single contact record using a unique identifier such as email or account number to verify existence or retrieve current field values. query contacts (with filter) In Connect, use the contacts query with a filter on email or a custom key attribute.
REST
Get Rich Content		 Retrieve dynamic/rich content blocks associated with mobile or in-app messages for rendering or auditing purposes. Not Available Rich content is managed within Campaign; no direct Connect equivalent for mobile rich content retrieval.
REST
Create Rich Content		 Programmatically create rich content blocks (e.g., carousels, banners) for mobile in-app or push messages. Not Available Rich content creation is Campaign-specific; Connect focuses on audience and data management.
REST
Get Published Messages		 Retrieve a list of published messages to maintain an up-to-date catalog in CRM or external systems. query messages (Early Access) Connect's Get Messages API returns metadata (personalization variables, links, status) but not message bodies or assets.
REST
In-App Content API		 Deliver and manage in-app content for mobile applications, including targeted banners or modals. Not Available In-app content delivery is handled by Campaign's mobile SDK infrastructure.
REST
Push to Segment		 Send mobile push notifications to a defined segment of contacts. Not Available Push delivery is Campaign-specific. Connect manages the audience; Campaign handles delivery.
REST
Push to Segment (Inline Content)		 Send push notifications to a segment with message content provided inline in the API request. Not Available Same as Push to Segment — delivery layer remains in Campaign.
REST
BCC REST API		 Automatically BCC a designated address on outgoing emails for compliance or archiving purposes. Not Available BCC functionality is specific to Campaign's email send infrastructure.
REST
Add Column to Relational Table		 Extend an existing relational table schema by adding a new column to support additional data attributes. mutation manage ReferenceSet Attributes In Connect, use the Manage Reference Set Attributes mutation to add new attributes to an existing reference set.
REST
Map Universal Behaviors		 Map behavioral event data to universal behavior definitions for use in segmentation and automation. Signals (real-time import) Connect's real-time signal import supports behavioral event ingestion. Mapping configuration may differ.
REST
Delete List		 Programmatically delete auto-generated queries or lists that are no longer needed, to keep the account clean. Not Available Segment/list deletion is UI only in Connect. Segments are calculated in real-time and do not require manual cleanup in the same way.
REST
SMS External Consent		 Update SMS consent status for contacts based on external opt-in/opt-out events captured outside Acoustic. mutation updateContacts (consent) Use Connect's consent mutation to update SMS channel consent. Ensure SMS is configured as a consent channel in your Connect subscription.
REST
Export SMS Messages		 Export a history of SMS messages sent via Campaign for reporting or CRM sync. mutation ExportSignals Signal exports via SFTP can include SMS send/delivery events depending on signal configuration.
REST
Send SMS to Contact Source		 Trigger an SMS send to a defined contact source or list. Not Available SMS delivery is handled by Campaign. Connect manages audience data only.
REST
Send Transactional SMS		 Send one-to-one transactional SMS messages triggered by real-time events (e.g., order confirmations, alerts). Not Available Transactional SMS delivery is Campaign-specific functionality.
REST
Check Status of SMS Sent		 Poll the delivery status of a transactional SMS message to confirm delivery or identify failures. query dataJob For job-based operations, use dataJob query. No direct SMS status check equivalent; signal exports may carry delivery events.
REST
Channel Consent API		 Retrieve or update a contact's consent status across multiple channels (email, SMS, push). mutation updateContacts (consent) Connect's Consent API supports per-channel and per-consent-group consent updates.
REST
Contact by Channel		 Look up a contact using a channel-specific identifier such as a phone number or push token. query contacts (with filter) Use addressable attributes (email, phone) as filter criteria in the Connect contacts query.
REST
Establish Identity		 Associate an anonymous user or device with a known contact identity for cross-channel tracking. Not Available Identity resolution is handled at the Campaign/SDK layer. Connect stores resolved contact records.
REST
Push to Contact		 Send a push notification to a specific known contact across their registered devices. Not Available Push delivery is Campaign-specific. Connect does not manage push delivery.
REST
Push to Device		 Send a push notification to a specific device token. Not Available Device-level push delivery is Campaign-specific.
REST
Send MO Message		 Simulate or relay a mobile-originated (MO) inbound SMS message for testing or workflow triggering. Not Available MO messaging is Campaign-specific SMS infrastructure functionality.
XML APIs
Automation
XML
GetProgramsByContact		 Retrieve all automation programs a contact is currently enrolled in, to understand their journey state in CRM. Not Available Connect does not have a program/journey enrollment query. Journey management remains in Campaign.
XML
GetPrograms		 Retrieve a list of all automation programs configured in the account for inventory or orchestration purposes. Not Available Program/journey definitions are Campaign-specific.
Database Management
XML
AddRecipient		 Add a new contact to a Campaign database, typically triggered by a CRM event such as a new account creation. mutation createContacts Direct equivalent. Use createContacts in Connect to add new contacts to the audience.
XML
CalculateQuery		 Validate the expected contact count from an auto-generated query before scheduling a mailing. Not Available Segments in Connect are calculated in real-time; there is no need to pre-calculate query counts.
XML
CreateQuery		 Auto-generate a query containing a specific list of contact account numbers to target with a mailing, based on CRM logic. mutation createImportJob Use a JSON object with createImportJob to import a contact set and create a segment upon import. Segments are real-time in Connect.
XML
DoubleOptInRecipient		 Trigger a double opt-in confirmation flow for a contact to verify email consent. Not Available Double opt-in workflow is managed within Campaign. Connect stores resulting consent status via the Consent API.
XML
ExportList		 Export contact records from Campaign databases to synchronize data with CRM, including opted-out contacts and new opt-ins within a date range. query contacts Use the contacts query with date-range filters. For OptedOutDate/OptedOutDetails equivalents, use the Connect Consent API. Confirm date-range filtering approach given ~1M contact volumes.
XML
GetLists		 Retrieve a list of all databases and auto-generated queries to identify and clean up obsolete items. query dataSets Use dataSets to retrieve audience and reference set metadata. Auto-generated query equivalents are not applicable in Connect's real-time segment model.
XML
GetListMetaData		 Retrieve schema details of a database (field names, types) to validate mapping before import or export operations. query dataSets The dataSets query returns audience structure including contact attribute names, types, and IDs needed for mutations.
XML
ImportList		 Bulk add or update master list contacts from a map file and source file via FTP. Used for daily refreshes and full monthly reloads of master contact lists. mutation createImportJob Direct equivalent. Supports CSV import. Use in combination with SFTP file upload to Connect's FTP directory.
XML
SelectRecipientData		 Retrieve current field values for a specific contact to verify data or pre-populate CRM records. query contacts (with filter) Use contacts query filtered by email or unique key. Returns all contact attributes.
XML
RemoveRecipient		 Remove duplicate contact records after an account merge in CRM to maintain data integrity. mutation deleteContacts Max 25 contacts can be deleted per request in Connect. For large-scale deduplication, batch requests accordingly. Action is irreversible — consider opt-out instead where appropriate.
XML
SetColumnValue		 Update a single field value for a contact record, such as flagging a preference or updating a status field. mutation updateContacts Use updateContacts with the specific attribute to update. Supports partial updates.
XML
UpdateRecipient		 Update contact attributes such as AccountNumber after a CRM merge/split, or update profile fields from CRM. mutation updateContacts Direct equivalent. Also use updateContacts (consent) variant to opt contacts back in when needed.
XML
OptOutRecipient		 Opt out a contact from email communications based on an unsubscribe event captured in CRM or an external system. mutation updateContacts (consent) Use the consent mutation to set the contact's channel consent status to opted-out.
XML
PurgeData		 Delete all records from a database or relational table, typically to reset staging tables before a fresh import. mutation deleteContacts / Update Reference Set For contacts, use deleteContacts (batched, max 25 at a time). For reference sets, re-import with replace action via createImportJob.
Contact List Management
XML
AddContactToContactList		 Add a contact to a specific contact list for targeted mailings or suppression lists. Not Available Contact lists are Campaign-specific. In Connect, segmentation is done via real-time audience filters rather than static lists.
XML
CreateContactList		 Create a new contact list to use as a mailing target or suppression group. Not Available Static contact lists are a Campaign concept. Connect uses dynamic real-time segments.
XML
AddContactToProgram		 Enroll a contact into an automation program to trigger a journey based on a CRM event. Not Available Program enrollment is Campaign-specific. Connect does not manage journey enrollment.
XML
GetContactMailingDetails		 Retrieve details of mailings sent to a specific contact for CRM communication history. mutation ExportSignals Signal exports include per-contact send/open/click event data. Filter by contact identifier after export.
Relational Table Management
XML
CreateTable		 Create a new relational table to store supplemental data associated with contacts (e.g., order history, product data). mutation createReferenceSet (via import) In Connect, reference sets serve the same purpose as relational tables. Create via import using createImportJob.
XML
ImportTable		 Replace relational table data nightly with a fresh file (map file action=replace) to keep supplemental data current. mutation createImportJob (reference sets) Use createImportJob with a replace action to refresh reference set data. Combine with SFTP file upload.
XML
ExportTable		 Export relational table data for auditing, backup, or synchronization with external systems. query referenceSetRecords Use referenceSetRecords query to retrieve reference set content. Supports filtering and pagination.
XML
PurgeTable		 Clear all records from a relational table prior to a full replacement import. mutation createImportJob (replace) Use a replace-mode import to effectively purge and reload a reference set in one operation.
XML
JoinTable		 Associate a relational table with a contact database to enable merge-field access and segmentation on relational data. mutation connectReferenceSet Use connectReferenceSet to link a reference set to the audience, enabling use of reference set attributes in segments and sends.
XML
InsertUpdateRelationalTable		 Add new records or update existing ones in a relational table, such as appending new order records. mutation updateReferenceSet Use updateReferenceSet to upsert records in a reference set.
XML
DeleteRelationalTableData		 Remove specific records from a relational table, such as expired or cancelled order records. Not Available Selective record deletion from reference sets is not currently available via API. Use replace-mode import to refresh the full dataset.
XML
DeleteTable		 Delete an entire relational table that is no longer needed. mutation deleteReferenceSet Use deleteReferenceSet to remove a reference set entirely from Connect.
Scoring Models
XML
GetContactScores		 Retrieve scoring model scores for a contact to use in CRM-based segmentation or next-best-action logic. Not Available Scoring models are Campaign-specific. Scores can be exported as contact attributes if synced to Connect.
XML
GetScoringModels		 Retrieve a list of all scoring models configured in the account. Not Available Scoring model management is Campaign-specific functionality.
Template & Mailing Management
XML
GetMailingTemplates		 Maintain an up-to-date list of active mailing templates in CRM, including mailing ID, subject, and name, for use in email series creation. query messages (Early Access) Connect's Get Messages API returns metadata (personalization variables, embedded links, message status) but not message bodies or assets.
XML
ScheduleMailing		 Automatically schedule mailings using auto-generated queries tied to CRM segmentation logic. Not Available Mailing scheduling is Campaign-specific. Connect provides the audience data; Campaign executes the send.
XML
SendMailing		 Trigger an immediate email send to a contact or list, used for transactional or event-driven messages. Not Available Email send execution is Campaign-specific. Connect does not send messages.
XML
ForwardToFriend		 Allow a recipient to forward a mailing to another contact via a tracked link. Not Available Forward-to-friend is a Campaign email feature with no Connect equivalent.
XML
PreviewMailing		 Generate a preview of a mailing for a specific contact to validate personalization before scheduling. Not Available Mailing preview is Campaign-specific.
XML
ExportMailingTemplate		 Export a mailing template's HTML and metadata for backup, auditing, or migration to another system. Not Available Template export is Campaign-specific. Connect's messages query provides metadata only (Early Access).
XML
SaveMailing		 Programmatically create or update a mailing template, used in automated template generation workflows. Not Available Template creation/editing is Campaign-specific.
XML
GetMessageGroupDetails		 Retrieve details about an automated message group to understand the messages associated with a triggered campaign. query messages (Early Access) Partial equivalent — Connect's messages query returns message metadata. Full group structure may require Campaign API.
Dynamic Content
XML
AddDCRuleset		 Create a new dynamic content ruleset to define content variation logic based on contact attributes. Not Available Dynamic content rulesets are Campaign-specific. Connect provides the audience attributes used in rules.
XML
DeleteDCRuleset		 Remove an outdated or unused dynamic content ruleset from the asset library. Not Available Campaign-specific.
XML
ExportDCRuleset		 Export a dynamic content ruleset for backup or migration. Not Available Campaign-specific.
XML
GetDCRuleset		 Retrieve details of a specific ruleset to audit or replicate dynamic content logic. Not Available Campaign-specific.
XML
ImportDCRuleset		 Import a previously exported ruleset into a Campaign account, used during migrations or environment promotions. Not Available Campaign-specific.
XML
ListDCRulesetsForMailing		 List all dynamic content rulesets applied to a specific mailing. Not Available Campaign-specific.
XML
ReplaceDCRuleset		 Replace an existing ruleset with an updated version as content strategy evolves. Not Available Campaign-specific.
XML
ValidateDCRuleset		 Validate a ruleset before applying it to a mailing to catch configuration errors. Not Available Campaign-specific.
Reporting
XML
ListRecipientMailings		 Retrieve the email history for a specific contact to build communication history in CRM. mutation ExportSignals Signal exports provide per-contact event history (send, open, click, opt-out). Filter post-export by contact ID.
XML
GetReportIdByDate		 Extract report IDs for emails sent within a date range as a precursor to pulling detailed metrics. Not Available No direct equivalent. Signal exports cover event-level data. Aggregate reporting remains in Campaign.
XML
GetSentMailingsForOrg		 Retrieve all mailings sent across the organization to maintain a complete send calendar in CRM. mutation ExportSignals Signal exports include message name, subject line, from/reply-to address, and personalization variables per send event.
XML
GetSentMailingsForUser		 Retrieve mailings sent by a specific Campaign user for user-level activity reporting. Not Available User-level send attribution is not available in Connect signal exports.
XML
GetSentMailingsForList		 Retrieve all mailings sent to a specific list or database, used to interpret raw recipient data and build CRM communication history. mutation ExportSignals Signal exports cover send events at the contact level. Segment names TBD for inclusion in Signal Export.
XML
GetAggregateTrackingForUser		 Retrieve aggregate email metrics (opens, clicks, bounces) for a specific user's sends. Not Available Aggregate reporting is Campaign-specific. Signal exports provide raw event data for custom aggregation.
XML
GetAggregateTrackingForOrg		 Retrieve organization-wide aggregate email performance metrics for executive reporting. Not Available Aggregate reporting is Campaign-specific.
XML
RawRecipientDataExport		 Export raw contact-level event data (clicks, opens, opt-outs) daily to build communication history in CRM and drive targeting segmentations. mutation ExportSignals Files dropped to SFTP. Clicked link URLs may be added to Signal Export (TBD). Segment names in Signal Export also TBD.
XML
WebTrackingDataExport		 Export web behavior tracking events for contacts to enrich CRM profiles with on-site activity. Signals (real-time import / export) Web behavior can be ingested as signals into Connect in real time and exported via ExportSignals.
XML
GetAggregateTrackingForMailing		 Retrieve aggregate performance metrics for a specific mailing (opens, clicks, bounces, unsubscribes). Not Available Mailing-level aggregate reporting remains in Campaign. Raw event data available via Signal Export.
Miscellaneous
XML
DeleteJob		 Delete a completed or failed data job to keep the job queue clean. Not Available Job lifecycle management is handled differently in Connect; no explicit DeleteJob equivalent.
XML
GetFolderPath		 Retrieve the folder path of a Campaign asset or object for organizational or migration purposes. Not Available Connect does not use a folder-based asset structure.
XML
GetJobStatus		 Monitor the status of a data job (import, export) to know when it completes before retrieving a file or continuing processing. query dataJob No formal documentation but accessible via GraphQL introspection in Connect.
